# Migrating from npm to Bun

This guide walks you through switching this SvelteKit project from npm to Bun.

## Why Bun?

- Faster package installs
- Drop-in npm replacement for most use cases
- Native TypeScript support
- Generates its own lockfile (`bun.lockb`)

## Steps

### 1. Remove npm-specific files

Delete the npm lockfile:

```bash
rm package-lock.json
```

Optionally remove `.npmrc` (Bun doesn't use it):

```bash
rm .npmrc
```

### 2. Install dependencies with Bun

```bash
bun install
```

This will:
- Install all packages from `package.json`
- Generate a `bun.lockb` lockfile (Bun's equivalent of `package-lock.json`)

### 3. Verify it works

Test the dev server:

```bash
bun run dev
```

Test the build:

```bash
bun run build
```

### 4. Commit the changes (optional)

If you use git:

```bash
git add -A
git commit -m "Switch from npm to bun"
```

## Notes

- All existing scripts (`dev`, `build`, `preview`, etc.) work the same way
- No code changes required - SvelteKit + Vite are fully compatible
- `bun.lockb` is already gitignored via the `.gitignore` pattern (it's a binary file)
- If you encounter any issues, delete `node_modules` and run `bun install` again

## Reverting to npm

If you need to switch back:

```bash
rm bun.lockb
npm install
```
